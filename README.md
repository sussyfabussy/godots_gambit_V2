# Godot's Gambit
**ITP Project 2026**

Godot's Gambit is a 2D strategy card game in which players must make tactical decisions while playing their deck to defeat opponents and achieve victory. This game was developed as part of the ITP course. This repository contains the source code, documentation, and workflow guidelines.

---

# Project Setup and Workflow

This document outlines the collaboration standards and the utilization of GitHub, Trello, and Google Sheets for this project.

## Prerequisites

Every team member is required to have the following accounts:
- GitHub Account: For version control and code reviews.
- Trello Account: For task management and project tracking.

## Project Organization

### 1. Task Management (Trello)
All project tasks are maintained as cards within Trello.
- Assignment: Every active card must be assigned to at least one team member.
- Integration: Use the GitHub Power-Up to link Trello cards to their corresponding branches or Pull Requests.

### 2. Prioritization System (P-System)
We utilize a numerical priority system. The urgency of a task decreases as the number increases:
- P1 Critical: Highest priority. These tasks require immediate attention (e.g., blockers or core functionality).
- P2 High: Important tasks required for the current milestone.
- P3 Medium: Standard tasks to be completed in the normal course of development.
- P4 and higher Low: Optional features, long-term backlog items, or "nice-to-have" enhancements.

### 3. Time Tracking (Google Sheets)
Work hours are documented manually in a centralized Google Sheets document.
- Document Link: https://docs.google.com/spreadsheets/d/1BTlpcZVNh4dfMMgLe4YJQCPOWJpAMAg-gPf-mKBoxUQ/
- Procedure: Team members must log their hours promptly (no later than the end of the working day).
- Required Data: Date, Reference to the Trello card, and time spent in hours.

### 4. Effort Estimates
We estimate the required effort for each task before starting development.
- Format: Estimates and Priorities are written directly into the Trello card title.
- Example: "P1: Combat Loop (30h)"
- Purpose: This allows us to see the priority, the task name, and the estimated workload at a single glance on the Trello board.

## GitHub Guidelines

1. Branching: Create a dedicated branch for every task (Format: feature/task-name).
2. Main Branch: Direct commits to the main branch are prohibited.
3. Pull Requests: Code changes must be merged via Pull Requests. Every PR requires at least one approval from another team member before merging.
